パス・アナライザーのマップ・タイプ¶

マップ タイプには、マップの構築に使用するデータが記述されます。ルール セット エディタから条件を使用してマップをフィルタリングできます。

新しいマップを作成するときに、マップの種類を選択します。既定では、次のマップ タイプが使用できます。

カテゴリー

説明

ページマップ

特定のインタラクション内でのページ訪問のシーケンスから作成されるマップ。ページマップにはデフォルトでいくつかのサブタイプがあり、特定のインタラクション属性に焦点を当てたページマップを作成することができます。

ゴールマップ

特定のゴールを変換することを中心に構成されたマップ。これは、選択したゴールにつながるインタラクションのマップを見たいユーザーにとって便利です。ゴール マップは、特定のゴールに到達した後に訪問されたページを無視します。

アセットマップ

PDF ファイルやビデオなどの特定のデジタル資産をダウンロードしたり、その資産と対話したりするためにコンタクトが取るパスを表示するように構成されたマップ。アセット マップでは、特定のアセットがダウンロードされた後に訪問されたページは無視されます。

キャンペーンマップ

特定のキャンペーンから来たコンタクトの経路体験を表示するマップ。

チャネルマップ

チャンネル マップを使用すると、特定のチャンネルからのすべてのインタラクションのページ マップを作成できます。

エクスペリエンスマップ

エクスペリエンスマップには、チャネル、キャンペーン、目標、イベント、成果などの他のインタラクション属性を含めることができます。また、より詳細なページマップよりも、より高度なパス分析に焦点を当てることができます。

成果マップ

結果マップを使用すると、特定の結果につながるページ マップを作成できます。

サイト固有のマップ¶

マップは、グローバルまたはサイト固有のものを使用できます。これは、マーケティング・コントロール・パネルの Path Analyzer のコンテンツ・ツリー内のマップの位置によって決まります。グローバル マップはすべて /Maps/Default maps/Global maps フォルダに、サイト固有のマップはすべて /Maps/Default maps/Site maps フォルダに配置されます。

カスタムマップ¶

カスタム マップは、デフォルトのマップ タイプに基づいていないマップです。カスタム マップは開発者によって作成され、/Maps/Custom maps フォルダに配置されます。
