##############################################
パターンマッチングの設定
##############################################

パターンカードは、特定のタイプのコンタクトの行動や興味をウェブサイトに反映させるために作成するカードです。パターンカードにプロファイル値を割り当て、それを使用して、さまざまなタイプの典型的なウェブサイトのアクティビティを識別します。パターンカードを使用して、パーソナライズルールを実装することもできます。

Sitecoreは、N-Dimensional Euclidean Distanceと呼ばれる測定値を使用して、コンタクトの蓄積されたプロファイル値がどのパターンカードに最も似ているかを計算し、マッピングする必要があります。

パターンマッチング機能を設定する場合、Sitecore が累積プロファイルスコアをパターンカードにマップする前に、コンタクトが何個のプロファイリングされたコンテンツアイテムとインタラクトする必要があるかを指定する必要があります。

パターンマッチングを設定するには、以下の手順に従います。

1. Sitecore build numberに移動して、Sitecore.Analytics.Tracking.configファイルを開きます。

2. configuration/sitecore/settings セクションで、次の行を追加します。

.. code-block:: xml

    <setting name="Analytics.Patterns.MinimalProfileScoreCount" value="3"/>


これは、コンタクトが3ページ以上リクエストした場合に、Sitecoreがコンタクトの行動をパターンカードにマッピングし始めることを意味します。

3. ページリクエストの適切な数だと思う値に変更してください。

   使用する値は、ウェブサイトの理解度、ウェブサイトのサイズ、プロファイル値を割り当てるページ数に基づいて設定する必要があります。
